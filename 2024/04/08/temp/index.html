第一章 计算机系统概述

一、名词解释

  1. 中央处理器（CPU）：P-27 通常把控制部件、运算部件和各类寄存器互联组成的电路称为中央处理器（CPU），简称处理器

  2. 算术逻辑部件（ALU）：P-26 用来进行算术逻辑运算的部件

  3. 通用寄存器：P-26 用来临时存放从主存取来的数据或运算的结果的寄存器

  4. 程序计数器（PC）：P-27 用来自动计算并保存下一条指令的地址的部件

  5. 指令寄存器（IR）：P-27 用来临时保存从主存中取出来的指令的寄存器

  6. 控制器：P-26 用于自动逐条取出指令并进行译码的部件被称为控制元件，也称控制器

  7. 主存储器：P-26 用来存放指令和数据的部件叫主存储器，也叫主存或内存

  8. 总线：P-27 通常把连接不同部件进行信息传输的介质称为总线

  9. 主存地址寄存器（MAR）：P-27 用来存放 CPU 送到地址线的主存地址的寄存器

  10. 主存数据寄存器（MDR）：P-27 用来存放发送到或从数据先取来的信息的寄存器

  11. 机器指令：P-30 由 0 和 1 组成，并且能为计算机理解和执行的指令被称为机器指令

  12. 指令操作码：

  13. 高级程序语言：P-30 高级程序设计语言简称高级程序语言，是面相算法设计的、较接近于日常英语书面语言的程序设计语言。如BASIC、C/C++、JAVA 等

  14. 汇编语言：P-30 用简短的英文符号和机器指令建立对应关系，以方便程序员编写和阅读的语言

  15. 机器语言：P-30 使用特定的计算机规定的指令格式而形成的 0/1 序列称为机器语言

  16. 机器级语言：P-30 汇编语言和机器语言都属于低级语言，它们统称位机器级语言

  17. 源程序：P-30 被翻译的程序称为源程序

  18. 目标程序：p-30 翻译生成的程序被称为目标程序

  19. **编译程序：**也称解释器，用来将源程序中的语句按其执行顺序逐条用子程序进行解释，并最终转换成机器指令执行

  20. 解释程序：P-30 也称解释器，用于将源程序中的语句按其执行顺序逐条翻译成机器指令并立即执行

  21. 汇编程序：P-30 也称汇编器，用来将汇编语言源程序翻译成机器语言目标程序

  22. 语言处理系统：P-35 提供程序编辑器和各类翻译转换软件的工具包统称为语言处理系统或软件的集成开发环境

  23. 设备控制器：P-34 外设通过 I/O 控制器或 I/O 适配器连接到主机上，I/O 控制器或 I/O 适配器统称为设备控制器

  24. 最终用户：P-36 使用应用软件完成特定任务的计算机用户称为最终用户

  25. 系统管理员：P-36 是指利用操作系统、数据库管理系统等软件提供的功能对系统进行配置、管理和维护， 以建立高效合理的系统环境供计算机用户使用的擦走人员

  26. 应用程序员：P-36 是指使用高级编程语言编制应用软件的程序员

  27. 系统程序员：P-36 是指设计和开发系统软件的程序员

  28. 指令集体系结构（ISA）：P-35 软件和硬件之间的界面就是指令集体系结构，简称指令集架构或指令系统

  29. 微体系结构：P-35 实现 ISA 的电路逻辑结构称为计算机组织或微体系结构

  30. 透明：P-37 在计算机技术中，一个存在的事物或概念从某个角度看似乎不存在，即对实际存在的事物或概念感觉不到，则称为透明

  31. 响应时间：P-40 是指从作业提交开始时间到作业完成所用时间

  32. 吞吐率：P-40 表示在单元时间内所完成的工作量

  33. 用户 CPU 时间：P-40 指真正用于运行用户程序代码的时间

  34. 时钟周期：P-40 节拍，计算机中最基本的、最小的时间单位。是时钟频率的倒数，也成为震荡周期

  35. 主频：P-40 cpu 的主频就是cpu 时钟信号的时钟频率，是 cpu 时钟周期的倒数，单位为 Hz

  36. CPI：P-40 表示执行一条指令所需要的时钟周期数

  37. 基准程序：P-43 专门用来进行性能评价的一组程序，能够很好地反映机器在运行实际负载时的性能，可

以通过在不用机器上运行相同的基准程序来比较在不同机器上的运行时间，从而评测其性能

  1. **MIPS：**每秒处理的百万级的机器语言指令数。这是衡量 CPU 速度的一个指标

  2. 峰值 MIPS：P-42 选取一组指令组合，使的得到的平均 CPI 最小，由此得到的 MIPS 就是峰值 MIPS

  3. 相对 MIPS：P-42 是根据某个公认的参考机型来定义的相应 MIPS 值，其值的含义是被测机型相对于参考机型 MIPS 的倍数

  4. MFLOPS：P-43 浮点操作速度的指标，表示每秒所执行的浮点运算有多少百万次

  5. GFLOPS：P-43 109 次/s

  6. TFLOPS：P-43 1012 次/s

  7. PFLOPS：P-43 1015 次/s

  8. EFLOPS：P-43 1018 次/s

  9. **指令：**用 0 和 1 表示的一串 0/1 序列，用来指示 cpu 完成一个特定的基本操作

  10. **翻译程序：**也称编译器，用来将高级语言翻译成汇编语言或机器语言目标程序

答：运算器、控制器、存储器、输入设备和输出设备

  • 运算器：是对数据进行运算的部件,其主要功能是对二进制数据进行算术运算(加、减、乘、除)和逻辑运算(与、或、非、移位),故又称为算术逻辑单元;

  • 控制器：是整个计算机的控制中心,其功能是控制计算机各个部件自动协调工作。

  • 存储器：用来存放程序和数据;

  • 输入设备：用来向计算机输入程序和原始数据;

  • 输出设备：用来输出计算机的处理结果及程序;

2. 什么是“存储程序”工作方式？ 答: (1) 事先编制程序 (2) 实现存储程序 (3) 自动、连续地执行程序

即事先编写程序,再由计算机把这些信息存储起来,然后连续地、快速地执行程序,从而完成各种运算过程。

6. 程序的 CPI 与哪些因素有关？

CPI = 程序总时钟周期数 / 程序总指令条数，CPI 与时钟周期、指令条数有关。在指令条数和时钟周期一定的情况下，CPI 越大，执行时间越长

7. 为什么说性能指标 MIPS 不能很好的反映计算机的性能？ 答：MIPS 反映了机器执行定点指令的速度，但是用 MIPS 来对不同的机器进行性能比较有时是不准确

或不客观的。因为不同的机器的指令集不同，指令的功能也不同。也许一个机器上一条指令的功能，在另外一个机器上要用多条指令来完成，这样，同样的指令条数所完成的功能可能完全不同。

8. 源程序到目标程序的过程叫什么？分别有哪几种类型？ 答：源程序到目标程序的过程叫 翻译

翻译程序分为：解释器（解释程序）、编译器（编译程序）、汇编器（汇编程序）

9. 指令由那几部分组成，什么决定了指令的操作类型？ 答：指令通常被划分成若干字段，有操作码和地址码

操作码：指出指令的操作类型，如取数、存数、加、减、传送、跳转等 地址码：指出指令所处的操作数的地址，如寄存器编号、主存单元编号等

第二章 数据的表示和运算

四、名词解释

  1. 真值：机器数真正的值（即现实世界中带有正负号的数）称为机器数的真值。

  2. 机器数：通常将数值数据在计算机内部编码表示的数称为机器数，机器数一定是一个 0/1 序列，通常缩写成十六进制形式。

  3. 数值数据：可用来表示数量的多少，可比较其大小，分为整数和实数。

  4. 非数值数据：没有大小之分，不表示数量的多少，主要包括字符数据和逻辑数据。

  5. 无符号整数：p-54 当一个编码的所有二进位都用来表述数值而没有符号位时，该编码表示的就是无符号整数。

  6. 带符号整数：p-55 又叫做有符号整数，它必须用一个二进制位来表示符号，其余位表示数值的编码就是带符号整数。

  7. 定点数：P-54 整数的小数点隐含在数的最右边，故无须表示小数点，因而也被称为定点数。

  8. 浮点数：

  9. 尾数：对于任意一个实数 X，可表示为：X=（-1）S*M*RE，M 是一个二进制定点小数，称为数 X 的尾数。

  10. 阶和阶码：对于任意一个实数 X，可表示为：X=（-1）S*M*RE，E 是一个二进制定点整数，称为数 X 的阶或指数；阶的移码表示称为阶码。

  11. 溢出：两个数进行数学运算后的结果超出了该数的表示范围被称为溢出。

  12. 规格化数：P-58 为了在浮点数运算过程中，尽可能多地保留有效数字的位数，使有效数字尽量占满尾数

数位，必须在运算过程中对浮点数进行“规格化”操作。

  1. 左规：P-58 左规时，尾数每左移一位，阶码减 1，直到尾数变成规格化形式为止。

  2. 右规：P-58 右规时，尾数每右移一位，阶码加 1，直到尾数变成规格化形式为止，右规时阶码会增加， 因此阶码有可能溢出。

  3. ASCII 码：P-63 ASCII 码即美国标准信息交换码。

  4. 汉字输入码：P-64 对每个汉字用相应的按键进行的编码表示就称为汉字的输入码，又称外码。

  5. 汉字内码：P-64 汉字被输入计算机内部后，就按照一种称为内码的编码形式在系统中存储、查找、传送等处理。

  6. 字长：P-65 在考察计算机性能时，一个真重要的性能参数就是计算机的字长，所谓字长通常是指 CPU

内部用于整数运算的数据通路的宽度。

  1. 大端方式：将数据的最高有效字节 MSB 存放在最小地址单元中，将最低有效字节 LSB 存放在最大地址单元中，即数据的地址就是 MSB 所在的地址。

  2. 小端方式：将数据的最高有效字节 MSB 存放在最高地址单元中，将最低有效字节 LSB 存放在低地址中， 即数据的地址就是 LSB 所在的地址。

  3. ALU：P-68 算术逻辑部件，用来完成基本的逻辑运算和定点数加减运算的部件。

  9. 流缓存区：P-258 文件 fd 的流缓存区 FILE 由缓冲区起始位置 base、下一个可读/写位置 ptr 以及剩下未读/写的字节数 cnt 来描述。

  10. 索引节点：

  11. 目录文件：P-264 目录本身是一种文件，称为目录文件。

  12. 目录项：

  13. 系统打开文件表：P-265 VFS 为系统中所有打开的文件维护了一张系统文件表，因此，该表也称为系统打开文件表。

  14. 磁盘高速缓存：P-267 传统的外部存储器是磁盘，因此上述高速缓存也称为磁盘高速缓存。

  15. 高速缓存 RAM：P-267 作为高速缓存的主存 RAM 区可称为高速缓存 RAM。

  16. I/O 控制方式：

  17. 程序直接控制 I/O：P-268 程序直接控制 I/O 方式的基本思想是直接通过查询程序来控制主机和外设的数据交换，因此也称为查询或轮询方式。

  18. 就绪状态：

  19. 中断控制 I/O：P-269 中断控制 I/O 方式的基本思想是，当需要进行 I/O 操作时，首先启动外设进行第一个数据的 I/O 操作，然后阻塞请求 I/O 的用户进程，并调度其他进程都 CPU 上执行，期间，外设在设备控制器的控制下工作。

  20. 中断屏蔽字：

  21. 多重中断：P-274 在中断处理过程中，若又到来了优先级更高的新中断请求，CPU 应立即暂停当前执行的中断服务程序，转去处理新终断，这种情况下称为多重中断。

  22. 中断嵌套：P-274 多重中断也称中断嵌套。

  23. DMA 方式：P-272

  24. DMA 控制器：P-272 直接存储器访问控制 I/O 方式使用专门的 DMA 接口硬件直接控制在外设和主存之间交换数据，数据不经过 CPU，把该接口硬件称为 DMA 控制器。

  25. I/O 接口：P-279：I/O 接口又称设备控制器或 I/O 控制器或 I/O 控制接口，或 I/O 模块，是介于外设和 I/O

总线之间的部分，不同的外设往往对应不同的设备控制器。

  1. 设备控制器：P-280 I/O 接口又称设备控制器，是连接外设和主机的“桥梁”，它在外设侧和主机侧各有一个接口。

  2. I/O 端口：P-280 通常把设备控制器中的数据缓冲寄存器、状态/控制寄存器统称为 I/O 端口。

  3. 控制端口：P-280 控制寄存器简称为控制端口。

  4. 数据端口：P-280 数据缓冲器简称数据端口。

  5. 状态端口：P-280 状态寄存器简称为状态端口。

  6. I/O 地址空间：P-280 所有 I/O 端口编号组成的空间称为 I/O 地址空间。

  7. 独立编址方式：P-281 独立编址方式对所有的 I/O 端口单独编号，使它们成为一个与主存地址空间独立的 I/O 地址空间。

  8. 统一编址方式：P-281 I/O 地址空间与主存地址空间统一编址，主存地址空间分出一部分地址给 I/O 端口编号。

  9. 存储器映射 I/O：P-281 将 I/O 端口映射到主存空间的某段地址，也称为存储映射方式。

  10. I/O 指令：P-280 设备控制在主机侧通过 I/O 总线和主机相联，CPU 可通过指令将控制信息写入控制寄存器、从状态寄存器读出状态信息或数据缓冲寄存器进行数据交换，通常把这类指令称为 I/O 指令。

  11. 可编程中断控制器：P-283 中断系统的功能一般通过可编程中断控制器实现。

  12. 中断请求寄存器：P-282 及时记录各种中断请求，通常用一个中断请求寄存器来记录。

  13. 中断响应优先级：P-283 中断响应优先权由中断查询程序或中断判优电路决定优先权，它决定多个中断同时请求时先响应哪个。

  14. 中断处理优先级：P-283 中断处理优先级则由各自的中断屏蔽字来动态决定，决定本中断与其他所有中断之间的处理优先关系。

  15. 自陷指令：P-248 又叫访管指令、软终端指令，用于实现在用户态下运行的进程调用操作系统内核程序， 即当运行的用户进程或系统实用进程欲请求操作系统内核为其服务时，可以安排执行一条陷入指令引起一次特殊异常。

  16. I/O 子系统：主要解决信息的输入和输出问题，即解决如何将所需信息通过不同外设输入到计算机中，或者计算机内部处理的结果如何通过响应的外设输出给用户。

  17. 运行时：运行时系统也称为运行时环境或简称运行时。一个运行时系统除了要支持语言基本的低级行为之外，还要实现更高层次的行为，如库函数等，甚至提供类型检查、调试以及代码生成与优化等功能。

  18. I/O 子系统主要包含 I/O 软件和 I/O 硬件两大部分。
